# BUG-001: Compile TypeError ('GeomType' object is not callable)

**Commit Hash:** `695635666d17650adf8bf170394d53e84e66cc63`
## Summary
A compile request fails with `TypeError: 'GeomType' object is not callable` under a specific condition involving the shapes a design script leaves in scope.

## Environment
- build123d 0.10.0 (in this version, `Shape.geom_type` is a property returning a `GeomType` enum)
- `intus_server.py` compile pipeline: `exec(code, env)` → iterate `env` to select export shapes → export STL

## Observed behavior
The compile route raises `'GeomType' object is not callable` and returns a failure response. No STL is produced.

## Trigger condition
The error occurs only when the executed script leaves a raw `bd.Shape` (e.g. a `Part` returned from a function) as a top-level variable. When the only top-level export-relevant variable is a `BuildPart`, the error does not occur.

## Relevant code path
In the shape-selection loop:

```python
for val in env.values():
    if isinstance(val, bd.Shape) and hasattr(val, "volume"):
        if val.geom_type() in ("SOLID", "COMPOUND"):   # line that raises
            shapes.append(val)
    elif hasattr(val, "part") and isinstance(getattr(val, "part"), bd.Shape):
        shapes.append(val.part)
```

- A top-level `BuildPart` is not a `bd.Shape`, so it is handled by the `elif` branch; the `if` branch is not entered.
- A top-level raw `bd.Shape` enters the `if` branch and reaches `val.geom_type()`.

## Mechanism
`val.geom_type` evaluates to a `GeomType` enum (a property access). The trailing `()` attempts to call that enum value, which raises the `TypeError`.

## Reproduction
1. Run a script through the compile route whose namespace contains a `BuildPart` at top level only → completes without this error.
2. Run a script whose namespace contains a raw `Part`/`Shape` at top level (e.g. `x = some_func_returning_part()`) → raises `'GeomType' object is not callable`.

## Notes / observations (not yet acted on)
- For `Solid` and `Compound` objects in this version, `geom_type` returns `GeomType.OTHER`; the comparison target `("SOLID", "COMPOUND")` consists of strings.
- The `if` branch had not been exercised by prior scripts, as none had left a raw `bd.Shape` at top level.
