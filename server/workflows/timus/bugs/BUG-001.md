# BUG-001: Timus Compiler TypeError ('GeomType' object is not callable)

**Commit Hash:** `ac70bd863311d6782858dc7432740a29756edab1`

## Summary
The `timus_server.py` encounters the exact same `TypeError: 'GeomType' object is not callable` bug as the Intus workflow. This happens when the SVG viewport generator attempts to parse top-level raw `bd.Shape` objects left in the environment by a complex script (like the `3x5shed` project).

## Mechanism
In `timus_server.py` around line 28, the `get_compound_from_code` function iterates through module variables:
```python
if val.geom_type() in ("SOLID", "COMPOUND"):
```
Just like in Intus, newer versions of `build123d` implement `geom_type` as a property that returns a `GeomType` enum. Calling it with `()` throws a `TypeError`. Additionally, complex solids and compounds may return `GeomType.OTHER`, which means checking against `"SOLID"` and `"COMPOUND"` strings will silently skip valid shapes.

## Reproduction
1. Run the `timus_server.py` SVG generation on a script that assigns a raw `bd.Part` or `bd.Shape` to a top-level variable (e.g. `single_portal = make_portal()`).
2. The server will catch the exception and return a simple red SVG with the error message: `Error: 'GeomType' object is not callable` (as seen in your screenshot).

## Proposed Fix
Update `get_compound_from_code` to properly check if `geom_type` is callable, and handle enum comparisons correctly, including the `OTHER` geometry type, identically to the fix implemented in the Intus workflow.
